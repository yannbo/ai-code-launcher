# AI Code Launcher

Native macOS launcher for jumping into a project folder and starting your coding tool with one click.

原生 macOS 启动器，用来快速进入项目目录并启动常用 AI 编程命令。

## Features / 功能

- Pick a single project folder or import all subfolders under a root directory
- Pin commonly used projects and remove folders you no longer need
- Reveal the current project directly in Finder
- Use one editable command dropdown for launch commands
- Built-in presets: `codex`, `claude code`, `opencode`
- Remember custom commands that you typed before
- Follow the macOS system language by default
- Support manual language switching inside the app

## Install / 安装

The repository includes two installable artifacts:

- `AI Code Launcher Installer.pkg`
- `AI Code Launcher.dmg`

仓库内已经包含两种可直接安装的产物：

- `AI Code Launcher Installer.pkg`
- `AI Code Launcher.dmg`

### `.pkg`

Double-click `AI Code Launcher Installer.pkg` and follow the installer.

双击 `AI Code Launcher Installer.pkg`，按安装向导完成安装。

### `.dmg`

Open `AI Code Launcher.dmg` and drag `AI Code Launcher.app` into `Applications`.

双击 `AI Code Launcher.dmg`，把 `AI Code Launcher.app` 拖到 `Applications`。

### First Launch / 首次打开

If macOS blocks the app because it is not notarized:

1. Right-click the app in Finder
2. Choose `Open`
3. Confirm once more

如果 macOS 因为没有公证而拦截应用：

1. 在 Finder 里右键应用
2. 选择 `打开`
3. 再确认一次打开

## Usage / 使用方式

1. Click `Choose Folder...`
2. Add only the current folder, or list all subfolders under it
3. Select the target project from the project list
4. Pick a preset command or type your own
5. Click `Launch`

1. 点击 `选择文件夹...`
2. 选择只添加当前目录，或者列出该目录下的所有子文件夹
3. 从项目列表中选中目标目录
4. 从命令下拉框中选择预置命令，或者直接输入自定义命令
5. 点击 `启动`

## Language / 语言

By default, the app follows the macOS system language. Open `AI Code Launcher > Preferences...` to manually switch between:

- `中文`
- `English`

应用默认跟随 macOS 系统语言。打开 `AI Code Launcher > 设置...` 后，你可以手动切换：

- `中文`
- `English`

You can also switch back to the system language from the same preferences window.

也可以在同一个设置窗口里恢复为“跟随系统语言”。

Language preference is saved in:

```text
~/Library/Application Support/ai-code-launcher/config.json
```

语言偏好也会保存在下面的配置文件里：

```text
~/Library/Application Support/ai-code-launcher/config.json
```

## Build / 构建

Build the `.app`:

```bash
zsh build_app.sh
```

Build the installer package:

```bash
zsh build_installer.sh
```

Build the `.dmg`:

```bash
zsh build_dmg.sh
```

Start the locally built app:

```bash
./start_launcher.command
```

## Project Layout / 项目结构

- `LauncherApp.m`: current native AppKit implementation
- `build_app.sh`: builds the `.app`
- `build_installer.sh`: builds the `.pkg`
- `build_dmg.sh`: builds the `.dmg`
- `generate_icon.py`: generates the app icon
- `assets/`: icon and packaging resources

- `LauncherApp.m`：当前使用中的原生 AppKit 主程序
- `build_app.sh`：构建 `.app`
- `build_installer.sh`：构建 `.pkg`
- `build_dmg.sh`：构建 `.dmg`
- `generate_icon.py`：生成应用图标
- `assets/`：图标和打包相关资源

Some earlier prototype files, such as `launcher.py` and `LauncherApp.swift`, are still kept in the repository but are not used by the current build flow.

仓库里还保留了一些早期尝试文件，例如 `launcher.py` 和 `LauncherApp.swift`，当前构建流程并不使用它们。

## Notes / 说明

- The current `.dmg` focuses on stable drag-and-drop installation
- The app is not signed or notarized yet
- The icon is generated by [generate_icon.py](generate_icon.py)

- 当前 `.dmg` 采用稳定优先的标准拖拽安装方式
- 应用目前还没有做签名和公证
- 应用图标由 [generate_icon.py](generate_icon.py) 生成
